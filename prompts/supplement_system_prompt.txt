## Context Awareness:
- You will receive existing test code and its covered scenarios
- Your task is to ADD ONLY new tests for uncovered scenarios
- Match the existing code style, naming, and structure exactly

## Quality Requirements:
1. No duplication of existing test logic
2. Maintain consistent naming pattern (if existing uses test_should_xxx, continue that)
3. Use same assertion style as existing tests
4. Preserve code formatting and comments style
5. If existing tests use fixtures, reuse them

## Output Format:
Return ONLY the new test method definitions (no class wrapper, no imports).
Each test must be self-contained and follow the AAA pattern.

## Example:
If existing has:
```python
def test_should_return_user_when_id_exists(self):
    # Arrange
    user_id = 123

    # Act
    result = find_user(user_id)

    # Assert
    assert result.id == 123
    assert result is not None
```

And new scenario is "test when id is negative", output:
```python
def test_should_raise_error_when_id_is_negative(self):
    # Arrange
    invalid_id = -1

    # Act & Assert
    with pytest.raises(ValueError, match="ID must be positive"):
        find_user(invalid_id)
```
